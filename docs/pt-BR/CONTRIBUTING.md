# Como contribuir

Obrigado por querer contribuir com o OpenSource for Humans.

Este projeto aceita contribuições de código, dados, design, pesquisa, documentação, tradução, acessibilidade, segurança, governança e parcerias.

## Regras essenciais

1. Não exponha dados pessoais de pessoas em situação de vulnerabilidade.
2. Não publique CPF, documento, endereço residencial, telefone privado ou relatos pessoais sensíveis.
3. Não prometa benefício, atendimento, vaga, abrigo, cura, proteção jurídica ou ajuda financeira.
4. Use linguagem simples, direta e acessível.
5. Informe fonte e data de verificação ao sugerir serviços.
6. Separe fato, estimativa e opinião.
7. Não use IA para publicar dados sem revisão humana.
8. Priorize acessibilidade e uso em celular.
9. Respeite os capítulos locais e o conhecimento comunitário.
10. Em caso de dúvida sobre privacidade, escolha não coletar, não publicar ou pedir revisão.

## Tipos de contribuição

## Código

Contribuições técnicas devem ter escopo claro, testes quando aplicável e não introduzir coleta desnecessária de dados.

## Dados

Dados de serviços devem conter fonte pública ou confirmação direta. Sugestões da comunidade não devem ser publicadas automaticamente.

## Design

Interfaces devem ser simples, acessíveis, mobile-first e adequadas a pessoas com baixa familiaridade digital.

## Tradução

Traduções devem preservar clareza e tom acolhedor. Evite termos jurídicos quando uma frase simples resolver.

## IA

Funcionalidades com IA devem seguir [AI_SAFETY_POLICY.md](AI_SAFETY_POLICY.md).

## Pull requests

Um bom pull request deve incluir:

- o que mudou;
- por que mudou;
- como foi testado;
- riscos de privacidade, segurança ou impacto social;
- screenshots quando houver mudança visual.

## Revisão

Mantenedores podem solicitar ajustes por qualidade técnica, privacidade, segurança, acessibilidade, clareza ou alinhamento com a missão.
